Recursively expands/unfolds container contents onto multiple lines.

This intention takes arrays and objects that are on a single line and expands them across multiple lines, with each item on its own line. It works recursively, so nested containers are also expanded. The intention respects your code style settings for comma usage.

Before (array):

items: [1, 2, 3]

After (array):

items: [
    1,
    2,
    3
]

Before (nested object):

config: {server: {host: "localhost", port: 8080}, debug: true}

After (nested object):

config: {
    server: {
        host: "localhost",
        port: 8080
    },
    debug: true
}

Code style: If your MAML code style is set to "Remove commas", the expanded output will use newlines for separation without commas. Otherwise, commas will be added between items.

Special behavior: When invoked on a key (rather than inside a container), this intention will expand the value's contents if the value is an array or object.

Note: This is the opposite of "Inline contents" which flattens items onto a single line.
