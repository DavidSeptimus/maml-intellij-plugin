Reports duplicate keys within MAML objects.

According to the JSON specification, object keys must be unique. While some parsers may accept duplicate keys, the behavior is undefined and typically results in the last occurrence overwriting previous values. This inspection helps catch these issues early.

Example: 

{
  name: "John",
  age: 30,
  name: "Jane"  // Duplicate key 'name'
}

Why duplicate keys are problematic:

  • Undefined behavior - the JSON specification does not define how parsers should handle duplicates
  • Data loss - most parsers silently discard all but the last value, losing earlier definitions
  • Confusion - readers may not realize which value is actually used
  • Maintenance issues - duplicate keys often indicate copy-paste errors or refactoring mistakes

How to fix:

  • Remove or rename duplicate keys to make them unique
  • If multiple values are intended, use an array instead
  • Merge the duplicate entries if they represent the same logical property

Note: This inspection compares the actual key names after removing quotes, so both "name" and name (if identifiers are allowed) are considered the same key.
