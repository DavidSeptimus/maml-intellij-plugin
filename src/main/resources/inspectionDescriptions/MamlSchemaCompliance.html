Reports MAML values that do not comply with their associated JSON schema.

This inspection validates MAML files against JSON schemas and highlights type mismatches, missing required properties, invalid enum values, pattern violations, min/max constraint violations, and other schema compliance issues.

Example: 

{
  name: "John",
  age: "thirty"  // Type mismatch: expected number, got string
}

JSON schemas can be associated with MAML files through:

  • JSON Schema mappings in Settings | Languages & Frameworks | Schemas and DTDs | JSON Schema Mappings
  • Schema URL in file comments
  • $schema property in the MAML file

The inspection supports all JSON Schema draft versions and provides comprehensive validation including:

  • Type validation - ensures values match expected types (string, number, boolean, object, array, null)
  • Required properties - verifies all required object properties are present
  • Enum validation - checks values are from allowed enum values
  • Pattern matching - validates strings against regex patterns
  • Numeric constraints - enforces minimum, maximum, multipleOf
  • String length - checks minLength and maxLength
  • Array constraints - validates minItems, maxItems, uniqueItems
  • Additional properties - enforces additionalProperties restrictions
  • Property dependencies - validates property dependencies
  • Schema composition - supports allOf, anyOf, oneOf, not
