Reports invalid identifier keys in MAML configuration files.

In MAML, keys can be either identifiers or quoted strings:

Identifier Keys

Identifier keys have specific character restrictions:

  • Must contain only letters (A-Z, a-z), digits (0-9), underscores (_), and hyphens (-)
  • Cannot be empty
  • Can be composed entirely of digits (e.g., 1234), but are always interpreted as strings

Quoted String Keys

Quoted string keys (delimited with ") follow the same rules as regular strings and allow a much broader set of characters. Empty quoted string keys are allowed.

Examples:

# Valid identifier keys
title: "Hello"
my-key: "value"
key_123: "test"
123: "numeric key"

# Invalid identifier keys
my.key: "value"        # Contains '.'
my key: "value"        # Contains space
key@home: "value"      # Contains '@'

# Valid quoted string keys (any characters allowed)
"my.key": "value"
"my key": "value"
"key@home": "value"
"": "empty key allowed"

The quick fix will automatically convert invalid identifier keys to quoted string keys.
