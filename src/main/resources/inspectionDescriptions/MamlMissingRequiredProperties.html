Reports MAML objects that are missing required properties defined in their associated JSON schema.

When a JSON schema specifies certain properties as required, all objects matching that schema must include those properties. This inspection identifies missing required properties and provides a quick fix to automatically add them with appropriate default values.

Example: 

{
  name: "John"
  // Missing required property 'email'
}

How to fix:

  • Use the quick fix "Add required property 'propertyName'" to automatically insert the missing property
  • Manually add the required property with an appropriate value

The quick fix will insert the missing property with a suggested default value based on the schema:

  • Schema default value if specified
  • First enum value if the property has enum constraints
  • Type-appropriate default (empty string for strings, 0 for numbers, false for booleans, etc.)

Schema configuration: JSON schemas can be associated with MAML files through Settings > Languages & Frameworks > Schemas and DTDs > JSON Schema Mappings
