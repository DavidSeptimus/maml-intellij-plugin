Reports invalid escape sequences in MAML string literals.

In MAML, single-line strings (delimited with double quotes ") support the following escape sequences:

  • \\ - backslash
  • \" - double quote
  • \n - newline
  • \r - carriage return
  • \t - tab
  • \b - backspace
  • \f - form feed
  • \/ - forward slash
  • \uXXXX - unicode escape sequence

Any other escape sequence (e.g., \x, \a) is invalid and will be highlighted as an error.

Examples:

# Valid
"Hello\nWorld"
"Tab\tseparated"
"Quote: \""

# Invalid
"Invalid\xescape"  # \x is not a valid escape
"Bad\aescape"      # \a is not a valid escape

The quick fix will escape the backslash to treat it as a literal character.

Note: Multiline strings (delimited with triple quotes """) do not require escape sequences and are not checked by this inspection.
